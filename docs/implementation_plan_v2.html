1. 文脈グループベースの感情分析

Backend + Frontend 優先度: 高

1.1 課題

現在は各台詞を個別に（前後数台詞のコンテキスト付きで）分析しているが、会話の「場面」や「話題の切り替わり」を検出できていない。同じ場面内の台詞群を1グループとして感情の文脈を共有すべき。

1.2 方針: 自動グループ検出 + ユーザー手動調整

以下の2段階で対応する:

Phase A: 自動グループ検出（バックエンド）

  • VoiceItem間のフレーム間隔（ギャップ）を基にグループを自動分割する
  • 閾値: 全VoiceItem間ギャップの中央値に基づく動的閾値、またはユーザー指定の固定フレーム数（デフォルト: 1フレーム以上の空白で分割）
  • YMM4編集時に「場面切替でVoiceItem間に1F以上の隙間を空ける」運用と連携
  • グループ内の全台詞テキストを結合して場面全体の感情コンテキストを生成

Phase B: ユーザー手動グループ編集（フロントエンド）

  • 分析結果一覧画面で台詞を範囲選択 → 「グループ化」「グループ分割」操作
  • グループ境界をドラッグで調整
  • グループ単位での一括プリセット変更

1.3 感情分析の改善フロー

VoiceItems → ギャップ検出 → 自動グループ分割 ↓ ユーザー手動調整（UI） ↓ グループごとに全台詞結合 → 場面コンテキスト生成 ↓ 各台詞を「場面コンテキスト + 個別台詞」で分析 ↓ 感情スコア出力

1.4 データ構造

    group_id: int

2. 感情強度・複合感情マッピングの拡張

Backend + Frontend 優先度: 高

2.1 課題

現在は感情6種類×1プリセットの単純マッピングのみ。感情の強度や複合感情に応じた表情の細分化ができない。

2.2 方針: 階層型マッピング（最大第3位まで）

マッピング構造

感情スコアを降順ソートし、第1位・第2位・第3位の組み合わせでパターンマッチングする。

# マッピング優先順位:
# 1. 三重複合感情 (第1位+第2位+第3位) — 設定されている場合のみ
# 2. 二重複合感情 (第1位+第2位) — 設定されている場合のみ
# 3. 単一感情 (第1位のみ)
# 4. デフォルト

# 複合感情の適用条件:
#   - 第1位のスコアが 65% (0.65) 以下
#   - 第2位以降もスコアが閾値以上
#   - ユーザーが該当パターンのプリセットを設定済み
#   → 未設定の場合は上位（二重→単一→デフォルト）にフォールバック

設定UIのツリー構造

喜 ─── [プリセット選択: 笑顔] ├─ 楽 ── [喜+楽: 笑顔小] │ └─ 驚き ── [喜+楽+驚き: びっくり笑顔] ├─ 驚き ── [喜+驚き: びっくり嬉しい] ├─ 照れ ── [喜+照れ: 照れ笑い] 怒 ─── [プリセット選択: むっ] ├─ 哀 ── [怒+哀: 悔しい] 哀 ─── [プリセット選択: 泣き] ├─ 怒 ── [哀+怒: 悔し泣き] 楽 ─── [プリセット選択: にこにこ] 驚き ── [プリセット選択: びっくり] ├─ 楽 ── [驚き+楽: きょとん笑顔] 照れ ── [プリセット選択: 照れ] 通常 ── [プリセット選択: 通常] ← 全感情が閾値以下の場合

2.3 データ構造

@dataclass
class CharacterConfig:
    preset_ini: str = ""
    tachie_dir: str = ""
    layer_offset: int = 1

    # 単一感情マッピング: {"joy": "笑顔", "anger": "むっ", ...}
    emotion_presets: dict[str, str]

    # 二重複合感情: {"joy+happiness": "笑顔小", "surprise+happiness": "きょとん笑顔"}
    compound_presets_2: dict[str, str]

    # 三重複合感情: {"joy+happiness+surprise": "びっくり笑顔"}
    compound_presets_3: dict[str, str]

    # パーツ単位オーバーライド (既存)
    emotion_parts: dict[str, dict[str, str]]

    # 複合感情適用の最大スコア閾値 (これ以下で複合判定)
    compound_max_score: float = 0.65

2.4 マッピングロジック（改修）

def map_emotion(emotion: EmotionResult, threshold: float = 0.3) -> str:
    active = emotion.above_threshold(threshold)
    if not active:
        return default_preset

    # スコア降順でソート → 第1位, 第2位, 第3位
    ranked = sorted(active.items(), key=lambda x: x[1], reverse=True)
    top_score = ranked[0][1]

    # 複合感情判定: 第1位が compound_max_score 以下
    if top_score <= compound_max_score and len(ranked) >= 3:
        key_3 = "+".join([r[0] for r in ranked[:3]])
        if key_3 in compound_presets_3:
            return compound_presets_3[key_3]

    if top_score <= compound_max_score and len(ranked) >= 2:
        key_2 = "+".join([r[0] for r in ranked[:2]])
        if key_2 in compound_presets_2:
            return compound_presets_2[key_2]

    # 単一感情: 第1位
    return emotion_presets.get(ranked[0][0], default_preset)

3. 個別台詞の表情手動変更

Backend + Frontend 優先度: 高

3.1 課題

自動マッピング結果に対して、ユーザーが個々の台詞の表情を手動で上書きできない。

3.2 方針

  • 感情分析結果一覧で台詞をクリック → サイドパネルに現在のプリセットと立ち絵プレビューを表示
  • プリセット選択ドロップダウンで別のプリセットに変更可能
  • パーツ単位の個別変更も可能（眉・目・口・体 等を個別ドロップダウンで選択）
  • 変更は overrides として管理し、再分析しても保持される

3.3 データ構造

# ユーザーによる個別上書き情報
# voice_index → override情報 のマッピング
overrides: dict[int, VoiceOverride]

@dataclass
class VoiceOverride:
    preset_name: str | None = None          # プリセット変更 (Noneなら自動のまま)
    part_overrides: dict[str, str] | None = None  # パーツ個別変更
    locked: bool = False                    # 再分析時に上書きを保持するか

3.4 対象ファイル

backend/app/main.py — 個別上書きAPIエンドポイント
frontend/src/components/DialogueList.tsx — 台詞クリック → 詳細パネル表示

4. 感情マッピング設定のツール内保存

Backend + Frontend 優先度: 中

4.1 課題

現在、感情マッピング設定はサーバーのインメモリ状態のみで保持されており、サーバー再起動で消失する。

4.2 方針

  • ツールのデータディレクトリ（backend/data/）にYAML形式で永続化
  • キャラクター名をキーとして設定を保存・読込
  • プロジェクト読込時に既存設定がある場合は自動ロード
  • UIに「設定を保存」「設定を読込」ボタンを配置
  • 設定のエクスポート/インポート機能（他環境への移行用）

4.3 保存パス

5. 立ち絵プレビュー表示の修正

Frontend 優先度: 中

5.1 課題

感情マッピング画面でプリセットのプレビュー（立ち絵パーツの合成表示）が正しく表示されていない。

5.2 方針

  • PresetPreview.tsx のPNG合成ロジックを修正
  • 描画順序を正しく適用: 他1→他2→他3→髪(半透明)→眉→目→口→髪→顔色→体→後1→後2→後3
  • バックエンドの /api/preset/image エンドポイントでPNGを配信 → フロントエンドで <img> タグで重ね合わせ
  • デフォルトパーツ（ItemParameter）とプリセットのオーバーライドパーツの両方を合成
  • 感情マッピング設定変更時にリアルタイムプレビュー更新

5.3 改修ポイント

  • プリセットで null のパーツはデフォルト（TachieDefaultItemParameter）のパーツを表示
  • 画像が存在しないパスの場合はスキップ（エラー非表示）
  • 髪パーツの2重描画（半透明＋通常）に対応

5.4 対象ファイル

frontend/src/components/PresetPreview.tsx — 合成ロジック修正
backend/app/main.py — デフォルトパーツ情報API追加

7. プロジェクト読込のD&D・ファイルダイアログ対応

Frontend 優先度: 低

7.1 方針

  • ドラッグ&ドロップ: ブラウザウィンドウへの .ymmp ファイルドロップでパス取得 → 読込API呼出
  • ファイルダイアログ: <input type="file"> で .ymmp ファイル選択

将来的にElectron化すればネイティブファイルダイアログとD&Dのフルパス取得が可能。現段階ではWeb UIの制約内でベストな体験を提供する。バックエンドに POST /api/project/upload エンドポイントを追加し、ファイル内容の直接アップロードにも対応する。

9. フレーム番号の時間表示変換

Backend + Frontend 優先度: 低

9.1 方針

  • 感情分析結果の F0, F100 等のフレーム番号を hh:mm:ss.SS 形式で表示
  • プロジェクトの VideoInfo.FPS（フレームレート）を使って変換
  • 変換式: seconds = frame / FPS → hh:mm:ss.SS にフォーマット
  • フロントエンド側のユーティリティ関数として実装
  • フレーム番号も併記: 00:03:24.17 (F6125)
