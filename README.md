# YMM4 EmotionMaker KIT

YMM4（ゆっくりムービーメーカー4）のプロジェクトファイルに含まれる台詞テキストを AI で感情分析し、キャラクターの表情（動く立ち絵のプリセット）を自動で配置するデスクトップアプリケーションです。

## 主な機能

- **BERT 感情分析**: 日本語に特化した WRIME v2 モデルによるローカル感情分析
- **LLM 感情分析**: Claude / OpenAI API による高精度分析（設定画面でモデル選択・API キー入力）
- **9 感情に対応**: 喜・怒・哀・楽・驚き・照れ・**嫌悪・恐れ・呆れ**（呆れは主に感情辞書で検出）
- **分析最適化ウィザード**: プロジェクト読み込み時に、動画の作りに合わせて分析設定（掛け合い／メリハリ／テンポと間／後処理）を最適化
- **文脈を考慮した分析**: 直前 N ターンを文脈として与え、話者名で文脈を区別。**文脈ギャップ**（無音区間の秒数）で場面ごとに区切って分析
- **reader ブレンド**: 書き手の感情と視聴者から見た感情の混合率を調整し、表現のメリハリをコントロール
- **感情辞書**: 台詞に特定の語句が含まれるとき、指定感情を強める（boost＝加算）／固定する（set）ルール
- **表情自動配置**: 分析結果に基づき、動く立ち絵の表情プリセットをタイムラインに自動配置（同キャラの立ち絵が表示されている区間に合わせて延長）
- **複合感情・強度別**: 「喜び＋照れ」など複合プリセット、単独感情の弱/中/強の強度別プリセットに対応。複合感情は**自動ミラー登録**で入れ替え順序（「喜+驚」→「驚+喜」、3感情は全6順列）へ一括登録（既定ON）
- **個人適応学習**: 台詞に正解感情をラベル付けして蓄積すると、分析が利用者のラベル傾向に適応（ローカルBERT、オフライン・プロジェクト横断で蓄積）
- **キャラの性格マップ**: キャラを「感情価×覚醒度」の2軸に配置し、感情の出やすさをキャラごとに方向付け（ソフトな事前分布）
- **感情後処理**: 「急変」「徐々」「減衰（余韻）」による表情の選び直し（場面の切れ目で余韻はリセット）
- **手動オーバーライド**: 個別の台詞に対して、感情で指定（クリック順の複合・強度）／プリセットで指定／パーツ個別変更／「前回の表情を保つ」を選択可能
- **アニメパーツ除外**: 眉・目・口のまばたき/口パク用フレーム（`名.0.png` や `名.a.png` 等）は候補から自動除外
- **動く立ち絵プレビュー**: PNG に加え GIF / WebP の立ち絵もプレビュー表示。ホイールでカーソル中心ズーム、ドラッグでパン
- **PSD立ち絵に対応**: PSD 規格の立ち絵（`.psd` ＋ 同梱の `-ymm.json` プリセット）を読み込み、サーバー合成したプレビューを表示。パーツ個別変更はレイヤーの表示・非表示で操作（クリックで表示切替／中クリックでソロ表示／Ctrl・Shift＋ホイールで兄弟グループ送り＝YMM4 互換）。表情の割り当ては従来どおりプリセット登録状況に依存
- **パーツ個別変更をプリセット化**: 「パーツ個別変更」で調整した状態に名前を付けて、YMM4 立ち絵プリセット（動く立ち絵の `preset.ini` ／ PSD の `-ymm.json`）の末尾へ追加登録。登録後はファイルを再読み込みしてアプリに即反映
- **設定自動保存**: キャラクターごとの感情マッピング・強度別/勾配プリセット・感情後処理設定を自動保存・復元
- **作業状態の保存/復元**: 設定・個別上書き・分析結果をまとめて `.ymmemo` ファイルに保存し、後から丸ごと復元
- **書き出し**: 既定で読み込んだプロジェクトと同じ場所に `_emotion` を付けて出力（保存ダイアログで変更可能、拡張子は常に `.ymmp`）

## 動作環境

- **OS**: Windows 10 / 11（64bit）
- **YMM4**: ゆっくりムービーメーカー4（動く立ち絵機能を使用するプロジェクト）
- **インターネット接続**: 初回の感情分析時に AI モデルを自動ダウンロードします（以降はキャッシュを使用）

> 配布版（インストーラー）はバックエンドを同梱しているため、**Python のインストールは不要**です。
> 以下はソースからビルド/開発する場合にのみ必要です。
> - **Python**: 3.10 以上（バックエンド／バンドルのビルド）
> - **Node.js**: 18 以上（フロントエンド／Electron のビルド）

## 使い方

1. **プロジェクト読み込み**: YMM4 プロジェクトファイル（.ymmp）をドラッグ＆ドロップ、またはパス入力で読み込み
2. **最適化ウィザード**: 読み込み直後に表示される設問に答えると、プロジェクトに合わせた設定で感情分析まで自動実行（「スキップ」で現在設定のまま分析。設定でON/OFF切替可）
3. **キャラクター設定**: アイコンでキャラを選び、各キャラの表情プリセット（preset.ini）を読み込み
4. **感情マッピング**: 9つの感情・複合感情・強度別（弱/中/強）にプリセットを割り当て（変更は自動保存）
5. **個別設定**: 台詞ごとに「感情で指定（複合・強度）」「プリセットで指定」「パーツ個別変更」「前回の表情を保つ」で上書き
6. **結果確認**: 右カラムの分析結果と左カラムの大型プレビューで配置を確認
7. **実行**: 出力先を確認（既定は `<元ファイル>_emotion.ymmp`）して表情アイテムを書き出し

感情辞書（語句→感情の補正）や文脈・reader ブレンド・後処理などの詳細は、右上の歯車アイコン → 設定 から調整できます。

### 個人適応学習（任意）

利用者のラベル傾向に分析を適応させる機能です（ローカルBERT時のみ）。

1. 初期画面の「**学習データ用として読み込み**」トグルを ON にして .ymmp を読み込む（解析せずラベリング画面が開く）
2. 各台詞に主感情を1つ付ける（キーボード `1–9`=感情 / `0`=中立 / `↑↓`=移動 / `Enter`=次へ）
3. 「ラベルを保存」→「学習を再構築」
4. **設定 → 感情分析 →「個人学習を使う」**を ON にし強度を調整。蓄積データはアプリ内に保存され、プロジェクトを跨いで貯まります。

### キャラの性格マップ（任意）

中央カラムの「キャラの性格マップ」でキャラを「感情価×覚醒度」の2軸に配置すると、その方向の感情が出やすくなります（強さスライダーで反映度を調整）。感情マッピングと同様にアプリ共通設定として保存されます。

### PSD立ち絵のプレビューとパーツ変更

PSD 規格の立ち絵（キャラに `.psd` ＋ 同じ階層の `<名前>-ymm.json` プリセット）を使ったプロジェクトは自動判別され、左カラムに**合成プレビュー**が表示されます。感情マッピングはプリセット名（`-ymm.json` の登録内容）に依存し、PNG 立ち絵と同じ操作で割り当てられます。

パーツ個別変更は**レイヤーの表示・非表示**で行います（YMM4 互換操作）。

- **クリック**：そのレイヤー単体の表示／非表示を切り替え
- **中クリック**：同じフォルダ内のそのレイヤーのみ表示（ソロ＝兄弟を非表示）
- **Ctrl または Shift＋ホイール**：同フォルダ内の表示状態を上下に1つ送り（端で停止）

調整した内容はプリセット基準との差分として保存され、感情で割り当てられるプリセットが変わっても追従します。書き出し時は YMM4 の PSD 表情アイテム（可視レイヤー集合）として出力されます。

### パーツ個別変更をプリセットとして保存

「パーツ個別変更」で調整した表情に名前を付けて、YMM4 立ち絵のプリセットとして追加登録できます（動く立ち絵＝`preset.ini`、PSD立ち絵＝`-ymm.json`）。登録は YMM4 と同じくファイルの**末尾に追記**され、登録直後に対象ファイルを再読み込みしてアプリのプリセット一覧／感情マッピングへ反映します。

> YMM4 はプリセット一覧を**起動中にメモリ保持**するため、YMM4 を起動したまま登録しても反映されず、YMM4 側の保存で上書きされる場合があります。**YMM4 を終了した状態で登録**してください。

作業を中断する場合はヘッダーの「💾 作業状態を保存」で `.ymmemo` ファイルに保存し、初期画面の「作業状態ファイルから復元」で再開できます。

詳しい使い方はアプリ内の「使い方」ボタン、設定は右上の歯車アイコンから確認できます。

│   │       └── llm_analyzer.py  # LLM API 分析
│   └── data/
│       └── config.yaml          # 設定ファイル（自動保存）
├── frontend/             # Next.js フロントエンド
│   └── src/
│       ├── app/                  # ページ
│       ├── components/           # UI コンポーネント
│       └── lib/                  # API クライアント
├── electron/             # Electron メインプロセス
│   ├── main.js                  # アプリケーションエントリ
│   └── preload.js               # プリロードスクリプト
├── scripts/
│   └── start.py                 # 開発用起動スクリプト
└── package.json          # Electron & ビルド設定

## 設定項目

`backend/data/config.yaml` で以下の設定を変更できます:

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| `emotion_model` | `local` | 感情分析モデル（`local` / `llm_claude` / `llm_openai`）。設定画面でも変更可 |
| `llm_api_key` | `""` | LLM 分析用の API キー（設定画面で入力。平文保存） |
| `emotion_threshold` | `0.3` | 感情判定の閾値（0.0〜1.0） |
| `context_turns` | `2` | 分析時に直前に含める発話ターン数（掛け合いの文脈） |
| `context_speaker_labels` | `true` | 文脈・対象を「話者名: 台詞」形式にして話者を区別 |
| `context_gap_seconds` | `0.4` | この秒数以上の無音で場面を区切る（プロジェクトの FPS でフレーム換算。0=1フレーム区切り） |
| `reader_weight` | `0.0` | 視聴者から見た感情(reader)の混合率（0=書き手 / 1=視聴者） |
| `personalization_enabled` | `false` | 個人適応学習（学習済みヘッド）の補正を有効化 |
| `personalization_strength` | `0.5` | 個人学習の補正の強さ（データ量でも自動調整） |
| `disabled_emotions` | `[]` | 検出を無効化する感情ラベル（自動OFFと連動） |
| `auto_disable_undetected` | `true` | 分析で検出されなかった感情ラベルを自動的にOFF |
| `show_optimizer_on_load` | `true` | プロジェクト読み込み時に最適化ウィザードを表示 |
| `extend_expression` | `true` | 表情を次の台詞まで延長するか |
| `max_gap_extend` | `300` | 最大延長フレーム数 |
| `postprocess_enabled` | `false` | 感情後処理（急変/徐々/減衰）の有効化 |
| `decay_rate` | `0.0` | 直前の感情を持ち越す減衰率（余韻） |
| `gradient_sudden_threshold` | `0.4` | 「急変」と判定する前行との差 |
| `gradient_gradual_window` | `3` | 「徐々」と判定する連続行数 |
| `gradient_gradual_max_delta` | `0.15` | 「徐々」と判定する 1 行あたりの上限変化量 |
| `ymm4_exe_path` | `""` | YukkuriMovieMaker.exe のパス（デフォルト立ち絵の取得に使用） |

> 多くの設定はアプリ内（感情マッピング・感情後処理・設定画面）で変更でき、自動的に `config.yaml` へ保存されます。手動編集も可能です。

## 更新履歴

### v1.0.11
- LLM での感情分析を文脈単位で行うように改修
- LLM の使用モデルの選択に DeepSeek (v4:デフォルト / Flash) を追加

### v1.0.10
- Claude API での感情分析が動作しない不具合を修正
- LLM の使用モデルを選択できる機能を追加（Claude Haiku / GPT-5.4 mini 等の軽量モデル・推論の深さ選択に対応）

### v1.0.9
- 感情分析にLLM感情分析を使用すると感情分析が行われない不具合の修正

### v1.0.8
- ライセンスを MIT から **AGPL-3.0-or-later** に変更（v1.0.7 までの版は MIT のまま）。アプリ内に「ライセンス」表記を追加
- YMM4 の「シーン機能」ごとの感情分析に対応（複数シーンをタブで切り替え、シーンごとに分析・配置。書き出しは解析済みの全シーンを一括反映）

### v1.0.7
- 「パーツ個別変更」の状態を YMM4 立ち絵プリセットとして名前付きで追加登録
- 複合感情の自動ミラー登録（「喜+驚」を登録すると「驚+喜」等にも自動登録）
- 設定画面の保存ボタンを下部に常駐
