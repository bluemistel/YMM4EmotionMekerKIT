# YMM4 EmotionMaker KIT

YMM4（ゆっくりムービーメーカー4）のプロジェクトファイルに含まれる台詞テキストを AI で感情分析し、キャラクターの表情（動く立ち絵のプリセット）を自動で配置するデスクトップアプリケーションです。

## 主な機能

- **BERT 感情分析**: 日本語に特化した WRIME v2 モデルによるローカル感情分析
- **LLM 感情分析**: Claude / OpenAI API による高精度分析（設定画面でモデル選択・API キー入力）
- **表情自動配置**: 分析結果に基づき、動く立ち絵の表情プリセットをタイムラインに自動配置
- **複合感情対応**: 「喜び＋恥ずかしさ」など複数感情の組み合わせプリセットに対応
- **感情後処理**: 「急変」「徐々」「減衰（余韻）」による表情の選び直し（有効/無効はアプリ全体で永続化）
- **手動オーバーライド**: 個別の台詞に対して表情・パーツを手動で上書き、または「前回の表情を保つ」を選択可能
- **アニメパーツ除外**: 眉・目・口のまばたき/口パク用フレーム（`名.0.png` や `名.a.png` 等）は候補から自動除外
- **動く立ち絵プレビュー**: PNG に加え GIF / WebP の立ち絵もプレビュー表示
- **プレビュー操作**: ホイールでカーソル中心ズーム、ドラッグでパン
- **プレビュー機能**: 配置結果をタイムラインでビジュアル確認
- **設定自動保存**: キャラクターごとの感情マッピング・勾配プリセット・感情後処理設定を自動保存・復元
- **作業状態の保存/復元**: 設定・個別上書き・分析結果をまとめて `.ymmemo` ファイルに保存し、後から丸ごと復元
- **書き出し**: 既定で読み込んだプロジェクトと同じ場所に `_emotion` を付けて出力（保存ダイアログで変更可能、拡張子は常に `.ymmp`）

## 動作環境

- **OS**: Windows 10 / 11（64bit）
- **Python**: 3.10 以上
- **Node.js**: 18 以上
- **YMM4**: ゆっくりムービーメーカー4（動く立ち絵機能を使用するプロジェクト）

## セットアップ

### 1. Python 依存パッケージのインストール

```bash
cd backend
pip install -e .
```

LLM 分析を使う場合（オプション）:

```bash
pip install -e ".[llm]"
```

### 2. Node.js 依存パッケージのインストール

```bash
# ルートディレクトリで
npm install
```

### 3. フロントエンドのビルド

```bash
npm run build:frontend
```

## 起動方法

### 開発モード

```bash
# バックエンドとフロントエンドを同時に起動
python scripts/start.py
```

### Electron アプリとして起動

```bash
# 開発モード
npm run dev

# ビルド（Windows インストーラー）
npm run build
```

ビルドすると `dist/` フォルダに NSIS インストーラー（`YMM4 EmotionMaker KIT Setup <version>.exe`）が出力されます。
インストーラーを実行するとインストール先の選択、デスクトップ／スタートメニューへのショートカット作成が行えます。
インストーラーを作らずに展開済みフォルダだけ確認したい場合は `npx electron-builder --win --dir`（`dist/win-unpacked/`）を使用します。

## 使い方

1. **プロジェクト読み込み**: YMM4 プロジェクトファイル（.ymmp）をドラッグ＆ドロップ、またはパス入力で読み込み（読み込むと自動で解析まで進みます）
2. **キャラクター設定**: 各キャラクターの表情プリセット（preset.ini）を読み込み
3. **感情マッピング**: 6つの基本感情・複合感情にプリセットを割り当て（変更は自動保存）
4. **感情分析**: 「再分析」ボタンで全台詞を再分析（分析中はローディング表示）
5. **個別設定**: 台詞ごとに表情・パーツを上書き、または「前回の表情を保つ」を選択
6. **プレビュー確認**: タイムラインで配置結果を確認
7. **実行**: 出力先を確認（既定は `<元ファイル>_emotion.ymmp`）して表情アイテムを書き出し

作業を中断する場合はヘッダーの「💾 作業状態を保存」で `.ymmemo` ファイルに保存し、初期画面の「作業状態ファイルから復元」で再開できます。

詳しい使い方はアプリ内の「使い方」ボタン、設定は右上の歯車アイコンから確認できます。

## 設定項目

`backend/data/config.yaml` で以下の設定を変更できます:

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| `emotion_model` | `local` | 感情分析モデル（`local` / `llm_claude` / `llm_openai`）。設定画面でも変更可 |
| `llm_api_key` | `""` | LLM 分析用の API キー（設定画面で入力。平文保存） |
| `emotion_threshold` | `0.3` | 感情判定の閾値（0.0〜1.0） |
| `context_window` | `3` | 前後の台詞を考慮する数 |
| `extend_expression` | `true` | 表情を次の台詞まで延長するか |
| `max_gap_extend` | `300` | 最大延長フレーム数 |
| `postprocess_enabled` | `false` | 感情後処理（急変/徐々/減衰）の有効化 |
| `decay_rate` | `0.0` | 直前の感情を持ち越す減衰率（余韻） |
| `gradient_sudden_threshold` | `0.4` | 「急変」と判定する前行との差 |
| `gradient_gradual_window` | `3` | 「徐々」と判定する連続行数 |
| `gradient_gradual_max_delta` | `0.15` | 「徐々」と判定する 1 行あたりの上限変化量 |
| `ymm4_exe_path` | `""` | YukkuriMovieMaker.exe のパス（デフォルト立ち絵の取得に使用） |

> 多くの設定はアプリ内（感情マッピング・感情後処理・設定画面）で変更でき、自動的に `config.yaml` へ保存されます。手動編集も可能です。
